# ユビキタス言語: ブロックエディタ

このドキュメントはプロジェクト内で使う共通用語を定義する。
コード・会話・ドキュメントすべてでこの用語を使用すること。

---

## ブロックの形状（Shape）


| 用語              | コード上の値         | 説明                                  | 例                          |
| --------------- | -------------- | ----------------------------------- | -------------------------- |
| **ハットブロック**     | `hat`          | スクリプトの起点。上にコネクタがなく、下にだけ繋がる。イベントトリガー | 「🏁が押されたとき」「スペースキーが押されたとき」 |
| **スタックブロック**    | `stack`        | 通常の命令ブロック。上下にコネクタがあり縦に連結する          | 「10歩動かす」「こんにちはと言う」         |
| **Cブロック**       | `c-block`      | 内部に他のブロックを囲む制御ブロック。ボディを1つ持つ         | 「もし〜なら」「10回繰り返す」           |
| **C-else ブロック** | `c-block-else` | ボディを2つ持つCブロック。分岐構造                  | 「もし〜なら / でなければ」            |
| **キャップCブロック**   | `cap-c`        | 下にコネクタがないCブロック。スクリプトの末端になる          | 「ずっと」(forever)             |
| **レポーターブロック**   | `reporter`     | 値を返す丸い形のブロック。スロットにはめ込む              | 「x座標」「変数の値」「1+2」           |
| **ブーリアンブロック**   | `boolean`      | 真偽値を返す六角形のブロック。ブーリアンスロットにはめ込む       | 「マウスが押された」「1>0」            |

## カスタム手続き（マイブロック）


| 用語             | コード上の値/型                 | 説明                                               |
| -------------- | ------------------------ | ------------------------------------------------ |
| **ディファインブロック** | `custom-define`          | カスタム手続きの定義ヘッダー。ハット形状。クリックで手続きエディタが開く             |
| **コールブロック**    | `custom-call`            | カスタム手続きの呼び出しブロック。スタック形状                          |
| **パラメータチップ**   | `param-chip`             | ディファインブロック**内部に埋め込まれている**引数名の表示部品。ブロックの一部であり独立しては存在しない。これをドラッグすると引数ブロックが新規生成される |
| **引数ブロック**     | `custom-argument`        | パラメータチップからドラッグして**ワークスペース上に生成される独立したレポーターブロック**。手続き実行時の引数値を返す。何個でも複製できる |
| **手続きエディタ**    | `ProcedureEditorPopover` | ディファインブロックをクリックすると開くポップオーバー。ラベル・引数の追加/削除/並び替えを行う |
| **カスタム手続き**    | `CustomProcedure`        | 手続きの定義情報。名前、トークン列（ラベルと引数の並び）、戻り値の有無を持つ           |

**パラメータチップと引数ブロックの関係:**
```
ディファインブロック: [ 挨拶する (名前) (回数) ]
                           ↑       ↑
                      パラメータチップ（ブロック内部の部品）
                           │
                     ドラッグすると
                           ↓
                      ┌────────┐
                      │  名前   │  ← 引数ブロック（独立したレポーター）
                      └────────┘
                      スロットにはめて使う
```

## ヘッダーレポーターコピー


| 用語               | コード上の型               | 説明                                                 |
| ---------------- | -------------------- | -------------------------------------------------- |
| **ヘッダーレポーターコピー** | `HeaderReporterCopy` | ハットブロック上に表示されるチップ。ドラッグするとそのイベントに紐づくレポーターブロックが生成される |


例: 「変数が変わったとき」ハットブロックの上に「新しい値」「古い値」チップがあり、ドラッグすると対応するレポーターブロックが作られる。

| ------------- | --------------- | -------------------------------- |

| **ブーリアンスロット** | `boolean-slot`  | ブーリアンブロックをはめ込む六角形の空きスロット         |

## 接続（Connection）

### コネクタの基本

コネクタはブロック間の接続点。**input（受け口）** と **output（差し込み口）** の2種類があり、output → input のペアで接続が成立する。

| 用語              | コード上のクラス/変数                | 説明                                          |
| --------------- | -------------------------- | ------------------------------------------- |
| **コネクタ**        | `Connector` (headless-vpl) | ブロック間の接続点。input（受け口）と output（差し込み口）のペアで接続する |
| **スナップ接続**      | `SnapConnection`           | スタックブロック同士のマグネット接続。近づけると自動的にスナップする          |


### コネクタ一覧

コネクタを指定するときは **指定名** を使う（例:「Repeatブロックの bodyEntry1」）。

| 指定名            | コード上の名前               | 型       | 説明                                                |
| -------------- | ---------------------- | ------- | ------------------------------------------------- |
| **topConn**    | `top`                  | input   | ブロック上部の受け口。上のブロックの bottomConn と接続する              |
| **bottomConn** | `bottom`               | output  | ブロック下部の差し込み口。下のブロックの topConn と接続する               |
| **valueConn**  | `value`                | output  | レポーター/ブーリアンブロックの右端の差し込み口。スロットに接続する              |
| **bodyEntry1** | `body-entry-1`         | input   | Cブロックのボディ1（条件成立側）の受け口。ボディ内の最初のブロックと接続する        |
| **bodyEntry2** | `body-entry-2`         | input   | C-elseブロックのボディ2（else側）の受け口。else内の最初のブロックと接続する   |
| **slotConn{N}** | `slot-connector-{N}`  | input   | ブーリアンスロットの受け口。N はブロック定義の入力インデックス                |

※ slotConn はブロック定義の inputs 配列に `boolean-slot` がある場合のみ生成される。


### ブロック形状ごとのコネクタ構成

| 形状               | topConn | bottomConn | valueConn | bodyEntry1 | bodyEntry2 | slotConn* |
| ---------------- | ------- | ---------- | --------- | ---------- | ---------- | --------- |
| **hat**          | —       | ✅          | —         | —          | —          | 定義次第      |
| **stack**        | ✅       | ✅          | —         | —          | —          | 定義次第      |
| **c-block**      | ✅       | ✅          | —         | ✅          | —          | 定義次第      |
| **c-block-else** | ✅       | ✅          | —         | ✅          | ✅          | 定義次第      |
| **cap-c**        | ✅       | —          | —         | ✅          | —          | 定義次第      |
| **reporter**     | —       | —          | ✅         | —          | —          | 定義次第      |
| **boolean**      | —       | —          | ✅         | —          | —          | 定義次第      |

「定義次第」= ブロック定義の inputs に `boolean-slot` がある場合のみ slotConn が生成される。


### コネクタ位置図解

#### hat（ハットブロック）

```
╭──────────────────────────╮
│  🏁が押されたとき          │
╰──┬───────────────────────╯
   bottomConn (output)
```

#### stack（スタックブロック）

```
   topConn (input)
╭──┴───────────────────────╮
│  10歩動かす                │
╰──┬───────────────────────╯
   bottomConn (output)
```

#### c-block（Cブロック — ボディ1つ）

```
   topConn (input)
╭──┴───────────────────────╮
│  10回繰り返す               │
│  ┌───────────────────────┤
│  │ bodyEntry1 (input)    │
│  │                       │
│  │   [ボディ1: ここに       │
│  │    ブロックをネスト]       │
│  │                       │
│  └───────────────────────┤
╰──┬───────────────────────╯
   bottomConn (output)
```

#### c-block-else（C-else ブロック — ボディ2つ）

```
   topConn (input)
╭──┴───────────────────────╮
│  もし <slotConn0> なら     │
│  ┌───────────────────────┤
│  │ bodyEntry1 (input)    │
│  │                       │
│  │   [ボディ1: 条件成立時]   │
│  │                       │
│  ├───── でなければ ─────────┤
│  │ bodyEntry2 (input)    │
│  │                       │
│  │   [ボディ2: else側]     │
│  │                       │
│  └───────────────────────┤
╰──┬───────────────────────╯
   bottomConn (output)
```

#### cap-c（キャップCブロック — 下部接続なし）

```
   topConn (input)
╭──┴───────────────────────╮
│  ずっと                    │
│  ┌───────────────────────┤
│  │ bodyEntry1 (input)    │
│  │                       │
│  │   [ボディ1: ここに       │
│  │    ブロックをネスト]       │
│  │                       │
│  └───────────────────────┤
╰──────────────────────────╯
   ※ bottomConn なし（末端）
```

#### reporter（レポーターブロック）

```
╭───────────────────╮
│  x座標             ├── valueConn (output)
╰───────────────────╯
```

#### boolean（ブーリアンブロック）

```
◇───────────────────◇
│  マウスが押された    ├── valueConn (output)
◇───────────────────◇
```


### 接続ペア

ブロック間の接続は常に **output → input** のペアで成立する。

#### 1. スタック接続（縦の連結）

```
╭──────────────────╮
│  Block A         │
╰──┬───────────────╯
   bottomConn (output)   ← A の差し込み口
   ↕  SnapConnection
   topConn (input)       ← B の受け口
╭──┴───────────────╮
│  Block B         │
╰──────────────────╯

ペア: A.bottomConn (output) → B.topConn (input)
```

#### 2. ボディ内ネスティング（Cブロック内への挿入）

```
╭──┴───────────────────╮
│  10回繰り返す           │
│  ┌───────────────────┤
│  │ bodyEntry1        │ ← 親の受け口（近接判定に使用）
│  │   ↕               │
│  │   topConn         │ ← 子の受け口
│  │ ┌─┴─────────────┐ │
│  │ │ 10歩動かす     │ │
│  │ └─┬─────────────┘ │
│  └───────────────────┤
╰──┬───────────────────╯

子ブロックの topConn が bodyEntry1 の AutoLayout に配置される。
bodyEntry1 は近接判定（ドラッグ時の吸着）に使用される。
```

#### 3. スロット接続（値ブロックのはめ込み）

```
╭──┴──────────────────────────╮
│  もし  ◇─────◇  なら        │
│        slotConn0 (input)    │ ← 親の受け口
│         ↕                   │
│        valueConn (output)   │ ← 子の差し込み口
│       ◇──────────────◇      │
│       │ マウスが押された │      │
│       ◇──────────────◇      │
╰──┬──────────────────────────╯

ペア: Boolean.valueConn (output) → If.slotConn0 (input)
```


### コネクタの指定例

コネクタを指定するときは `{ブロック名}.{コネクタ指定名}` の形式を使う:

```
Repeat.bodyEntry1     → Repeatブロックのボディ入口
IfElse.bodyEntry2     → If-Elseブロックのelse側ボディ入口
If.slotConn0          → Ifブロックの最初のブーリアンスロット
(x座標).valueConn     → レポーターブロックの値出力
Move.topConn          → Moveブロックの上部受け口
Hat.bottomConn        → ハットブロックの下部差し込み口
```

接続を指示するときは:
```
A.bottomConn → B.topConn           スタック接続
Boolean.valueConn → If.slotConn0   スロット接続
Stack.topConn → Repeat.bodyEntry1  ボディ内挿入
```

## ネスト構造


| 用語            | コード上の変数/型                    | 説明                                                                |
| ------------- | ---------------------------- | ----------------------------------------------------------------- |
| **ネストスロット**   | `nestedSlots`                | レポーター/ブーリアンブロックがスロットにはめ込まれている状態の記録。キー: `${blockId}-${inputIndex}` |
| **ネスティングゾーン** | `NestingZone` (headless-vpl) | ドラッグ中にブロックをドロップできる領域の定義                                           |
| **スロットゾーン**   | `SlotZoneMeta`               | レポーター/ブーリアンのスロットにブロックをはめる領域                                       |
| **ボディゾーン**    | `BodyZoneMeta`               | Cブロックの内部にブロックを入れる領域                                               |
| **チェーン**      | `collectConnectedChain()`    | スナップ接続で縦に繋がったブロック群。ドラッグ時はチェーン全体が一緒に動く                             |

## パレット


| 用語           | コード上のクラス/変数     | 説明                                                                                              |
| ------------ | --------------- | ----------------------------------------------------------------------------------------------- |
| **ブロックパレット** | `BlockPalette`  | カテゴリごとにブロック定義を一覧表示するパネル。ドラッグでワークスペースにブロックを追加する                                                  |

| **ブロック定義**   | `BlockDef`      | ブロックの型情報。形状・色・入力定義・カテゴリ・オペコードを持つ                                                                |
| **オペコード**    | `opcode`        | ブロックの一意な操作コード。ランタイムで命令を識別する                                                                     |

## レイアウト


| 用語            | コード上の型/定数                   | 説明                                                  |
| ------------- | --------------------------- | --------------------------------------------------- |
| **レンダー順**     | `containers` の配列順           | 後ろほど z-index が高い（手前に表示）。`bringChainToFront` で最前面に移動 |
| **Cブロック参照**   | `CBlockRef`                 | Cブロックのレイアウト情報。ボディレイアウト・エントリコネクタ・ボトムコネクタへの参照         |
| **オートレイアウト**  | `AutoLayout` (headless-vpl) | コンテナ内の子要素の自動配置。Cブロックのボディサイズ計算に使用                    |
| **スロットレイアウト** | `SlotLayoutRef`             | レポーター/ブーリアンスロットの位置・サイズ情報                            |

## シリアライズ


| 用語             | コード上の型                 | 説明                                                |
| -------------- | ---------------------- | ------------------------------------------------- |
| **プロジェクトデータ**  | `BlockProjectData`     | スプライト1つ分のブロックデータ。カスタム手続き + ワークスペースデータ             |
