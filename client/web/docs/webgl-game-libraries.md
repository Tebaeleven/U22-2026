# WebGL 対応ゲーム/レンダリングライブラリ比較

## 2D ゲームエンジン・フレームワーク

### Phaser

- **公式**: https://phaser.io/
- **概要**: ブラウザ2Dゲーム開発で最も人気のあるフレームワーク
- **強み**: シーン管理・物理エンジン（Arcade / Matter.js）・入力・オーディオ・タイルマップがオールインワン。公式サンプル1500+。Canvas フォールバックあり
- **弱み**: バンドルサイズが大きい（フルビルドで1MB超）。TypeScript の型が不完全な箇所あり。3D非対応
- **ライセンス**: MIT
- **メンテナンス**: 活発

### PixiJS

- **公式**: https://pixijs.com/
- **概要**: ブラウザ2Dレンダリング最速クラスのライブラリ
- **強み**: WebGL/WebGPU による高速スプライト描画。v8 で WebGPU 対応・アーキテクチャ刷新。フィルター（ブラー、グロー等）・マスク・ブレンドモードが豊富。他ライブラリと組み合わせやすい
- **弱み**: ゲームエンジンではないため物理・衝突判定・シーン管理は別途必要。v7→v8 で破壊的変更多い
- **ライセンス**: MIT
- **メンテナンス**: 活発

### Excalibur.js

- **公式**: https://excaliburjs.com/
- **概要**: TypeScript ファーストの2Dゲームエンジン
- **強み**: Actor ベース設計。衝突判定・物理・アニメーション・入力・オーディオ・タイルマップ内蔵。ドキュメントが丁寧で初心者に優しい
- **弱み**: パフォーマンスは Phaser や PixiJS にやや劣る。コミュニティが小さくプラグイン・サンプルが少ない
- **ライセンス**: BSD 2-Clause
- **メンテナンス**: 活発（v0.x 系）

### Kaplay（旧 Kaboom.js）

- **公式**: https://kaplayjs.com/
- **概要**: シンプルな API でゲームプロトタイピングが速い2Dエンジン
- **強み**: ECS 的コンポーネント設計。衝突判定・入力・シーン管理・オーディオ内蔵。初心者・ゲームジャムに最適
- **弱み**: 大規模ゲームには不向き。高度なレンダリング（シェーダー等）は限定的。Kaboom.js は開発終了、Kaplay に移行
- **ライセンス**: MIT
- **メンテナンス**: 活発

### MelonJS

- **公式**: https://melonjs.org/
- **概要**: 軽量な2Dゲームエンジン
- **強み**: Tiled マップエディタとの統合が優秀。物理・衝突判定・オブジェクトプーリング・パーティクル内蔵。ES6 モジュールベース
- **弱み**: コミュニティが小さい。Phaser と比べるとサンプル・ドキュメントが少ない
- **ライセンス**: MIT
- **メンテナンス**: やや低下傾向

### Kontra.js

- **公式**: https://straker.github.io/kontra/
- **概要**: 超軽量（コア 3KB gzip）のゲームマイクロライブラリ
- **強み**: js13kGames 向けに設計。ツリーシェイキング対応。スプライト・入力・衝突判定・タイルマップ等の基本機能あり
- **弱み**: 機能が最小限。主に Canvas 2D ベース（WebGL は限定的）。コミュニティが非常に小さい
- **ライセンス**: MIT
- **メンテナンス**: 低頻度

## 3D エンジン・ライブラリ

### Three.js

- **公式**: https://threejs.org/
- **概要**: ブラウザ3Dのデファクトスタンダード
- **強み**: 圧倒的なコミュニティとエコシステム。PBR・シャドウ・ポストプロセス・パーティクルなど豊富な機能。glTF/OBJ/FBX 等のローダー完備。WebGPU レンダラー開発中
- **弱み**: ゲームエンジンではないため物理・衝突判定・ECS 等は別途必要。API の破壊的変更が比較的多い。バンドルサイズが大きくなりがち
- **ライセンス**: MIT
- **メンテナンス**: 非常に活発

### Babylon.js

- **公式**: https://www.babylonjs.com/
- **概要**: Microsoft 開発のフル機能3Dエンジン
- **強み**: TypeScript ファースト。物理エンジン統合（Havok）・GUI システム・アニメーション・パーティクルすべて内蔵。Node Material Editor。WebGPU を早期からサポート。Playground 環境あり
- **弱み**: バンドルサイズが非常に大きい（数MB）。学習コストが高い。Three.js と比較するとコミュニティが少ない
- **ライセンス**: Apache 2.0
- **メンテナンス**: 非常に活発

### PlayCanvas

- **公式**: https://playcanvas.com/
- **概要**: クラウドエディタ付き3Dゲームエンジン
- **強み**: 軽量で高パフォーマンス（モバイル向けに最適化）。PBR・物理（Ammo.js）・アニメーション・パーティクル内蔵。WebXR 対応。Snap 傘下
- **弱み**: エディタの無料プランに制限あり。コードオンリーだとドキュメントがエディタ前提の記述が多い。コミュニティが比較的小さい
- **ライセンス**: エンジンは MIT / エディタは商用
- **メンテナンス**: 活発

### OGL

- **公式**: https://oframe.github.io/ogl/
- **概要**: 超軽量な WebGL ライブラリ（Three.js の 1/10 以下）
- **強み**: 低レベル API に近いが基本抽象化あり。WebGL の学習用途にも良い。Tree-shakeable
- **弱み**: 機能が最小限。ゲーム開発には機能不足
- **ライセンス**: MIT
- **メンテナンス**: メンテされている

### A-Frame

- **公式**: https://aframe.io/
- **概要**: HTML タグベースの宣言的 3D/VR シーン構築フレームワーク
- **強み**: Three.js 上に構築。WebXR（VR/AR）対応が簡単。ECS アーキテクチャ。豊富なコミュニティコンポーネント
- **弱み**: VR 以外の用途では Three.js を直接使う方が効率的。パフォーマンスチューニングが難しい
- **ライセンス**: MIT
- **メンテナンス**: コミュニティ主導

## 2D & 3D 両対応

### Cocos Creator

- **公式**: https://www.cocos.com/en/creator
- **概要**: Unity ライクなエディタ付きゲームエンジン
- **強み**: 2D/3D 両対応。物理（Bullet / Box2D）・UI・アニメーション完備。マルチプラットフォーム出力（Web, iOS, Android, デスクトップ）。中国市場で非常に強い
- **弱み**: エディタが必須に近い。英語圏のドキュメントが薄い。セットアップが重い
- **ライセンス**: エンジンは MIT
- **メンテナンス**: 活発

### Godot（HTML5 Export）

- **公式**: https://godotengine.org/
- **概要**: フル機能ゲームエンジンが WebAssembly でブラウザ動作
- **強み**: エディタ・物理・アニメーション・オーディオすべて内蔵。GDScript / C# で開発
- **弱み**: JavaScript ライブラリではなく WASM バイナリ出力。初期ロードサイズが大きい。DOM 連携が弱い
- **ライセンス**: MIT
- **メンテナンス**: 非常に活発

## クリエイティブ / ビジュアライゼーション寄り

### p5.js

- **公式**: https://p5js.org/
- **概要**: Processing の JavaScript 版。教育・アート・プロトタイピング向け
- **強み**: WEBGL モードで3D描画可能。非常にシンプルな API。巨大なコミュニティと教育リソース
- **弱み**: ゲーム開発には不向き（パフォーマンス・機能面）。大規模プロジェクトには設計が合わない
- **ライセンス**: LGPL 2.1
- **メンテナンス**: 非常に活発

### Two.js

- **公式**: https://two.js.org/
- **概要**: ベクターグラフィクス特化の2Dライブラリ
- **強み**: SVG / Canvas / WebGL 切替可能。アニメーション API が直感的。データビジュアライゼーション向き
- **弱み**: ゲームエンジンではない。WebGL モードでも PixiJS には及ばない
- **ライセンス**: MIT
- **メンテナンス**: 低頻度

### Regl

- **公式**: https://github.com/regl-project/regl
- **概要**: 関数型スタイルの WebGL ラッパー
- **強み**: WebGL のステート管理を自動化しつつ低レベルな制御を維持。非常に軽量
- **弱み**: シーングラフ等の高レベル抽象化なし。ゲーム開発には不向き
- **ライセンス**: MIT
- **メンテナンス**: 低頻度（API は安定）

## 物理エンジン（補足）

- **Matter.js** — 2D物理のデファクト。Phaser に統合済。Canvas 描画（WebGL ではない）
- **Cannon-es** — 3D物理。Three.js / Babylon.js と組み合わせて使用。MIT
